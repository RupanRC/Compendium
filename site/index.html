    04 / FLIGHT DOCTRINE — HOW I WORK

    Rules that keep
    systems survivable.

    Six operating principles, forged in defense programs and applied daily to AI delivery. Non-negotiable, every engagement.

    SPEC

    Spec before code

    Every feature starts as a written spec with numbered acceptance criteria and passes through approval gates. Aviation taught me that checklists are not bureaucracy — they are how complex systems stay survivable.

    TDD

    Tests are the contract

    TDD by default — EditMode suites in Unity, E2E in the browser, regression packs on platforms. Work is done when the suite is green, not when it looks done.

    HW

    Verify on real hardware

    A feature verified on a physical headset, tablet or deployed portal is worth ten verified on a dev machine. I close the loop on the device the user actually holds.

    AUTO

    Automate the meta-work

    Weekly reports, cost workbooks, pitch decks, WBS documents — all generated by tooling I built once. Human hours go to decisions, not formatting.

    GATE

    Autonomy with human gates

    I run fleets of AI agents, but every merge crosses a human approval gate with a tamper-proof audit trail. Trust the machine to work; verify before it lands.

    SHIP

    Velocity is a habit

    Nine feature updates in a day. Three phases of a platform rebuild in a weekend. Sustained, not heroic — the system makes the pace.
